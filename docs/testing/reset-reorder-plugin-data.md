# Reset: Reorder Plugin Data

This document describes the destructive reset script that removes all persisted data owned by the `reorder` plugin while leaving core Medusa store data untouched.

## Script

- [reset-reorder-plugin-data.ts](../../scripts/reset-reorder-plugin-data.ts)

## Purpose

Use this script when you want to return a Medusa store to a plugin-clean state without deleting:
- products
- variants
- customers
- orders
- carts
- other core Medusa records

This is useful for:
- local development resets
- QA cleanup between plugin test passes
- restoring a store before reseeding plugin scenarios

## What It Removes

The script deletes all rows from plugin-owned tables:
- `subscription`
- `renewal_cycle`
- `renewal_attempt`
- `dunning_case`
- `dunning_attempt`
- `cancellation_case`
- `retention_offer_event`
- `subscription_log`
- `subscription_metrics_daily`
- `plan_offer`
- `subscription_settings`

## What It Does Not Remove

The script does not delete core Medusa store data such as:
- `product`
- `product_variant`
- `customer`
- `order`
- `cart`
- auth and user records

## How to Run

Run the script from the root of your Medusa app:

```bash
cd my-medusa-store
npx medusa exec ../reorder/scripts/reset-reorder-plugin-data.ts
```

Adjust the relative path if your `reorder` plugin repository is located elsewhere.

## Important Warning

This script is intentionally destructive for plugin-owned data.

It removes:
- QA seed data
- manually created plugin records
- runtime data generated by the `reorder` plugin

It should not be used if you want to preserve any existing plugin state.
